# My Kid 3D Gallery — Design Spec
_Date: 2026-06-05_

## Overview

A browser-based 3D virtual art gallery for displaying kids' physical artwork digitally. Users walk through the gallery in third-person view, approaching framed artwork hung on 3D walls. Walking close to a piece triggers a fullscreen view. The goal is to preserve kids' artwork digitally so physical copies don't need to be kept.

---

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| 3D engine | Three.js (WebGL) | Lightweight (~600KB), full control, static-site compatible |
| Language | Vanilla JS + HTML | No framework needed — static files only |
| Character model | Low-poly humanoid .glb (Mixamo) | Fun, game-like, kids will enjoy seeing a character |
| Artwork registry | `manifest.json` | Auto-generated by `npm run scan` |
| Hosting | `jdpartners.co/games/my-kid-3d-gallery/` | Same git push workflow as existing games |

## Gallery Structure

### Lobby
- Starting point when the gallery loads
- Two doors side by side: left → Kid 1's Wing, right → Kid 2's Wing
- Wing names displayed on arches above each door
- Walk back through entrance to return to lobby from either wing

### Wings
- Each kid gets their own wing (series of connected rooms)
- Rooms are generated automatically from `manifest.json`
- Each room holds **6 artworks**: 3 on the left wall, 3 on the right wall
- A door at the far end of each room leads to the next room
- When a room is full and more artwork exists, a new room is created automatically
- Each wing has a distinct colour theme (configured per kid in `manifest.json`)

### Lighting
- Ambient light for base illumination
- Individual SpotLights above each artwork frame (spotlight cone pointing at painting)
- Subtle floor reflection under each spotlight

## manifest.json Format

```json
{
  "kids": [
    {
      "name": "Kid 1 Name",
      "color": "#70d0c0",
      "artworks": [
        {
          "file": "artwork/kid1/rainbow.jpg",
          "title": "",
          "date": "",
          "age": ""
        }
      ]
    },
    {
      "name": "Kid 2 Name",
      "color": "#d070c0",
      "artworks": [
        {
          "file": "artwork/kid2/dog.jpg",
          "title": "",
          "date": "",
          "age": ""
        }
      ]
    }
  ]
}
```

- `title`, `date`, `age` are all optional — leave blank, fill in later
- `color` sets the wing's accent colour (door frame, wall trim, spotlight tint)
- File paths are relative to the project root

## Adding Artwork Workflow

```
1. Scan and save artwork image (JPEG/PNG, resize to max ~1500px wide)
2. Drop image file into artwork/kid1/ or artwork/kid2/
3. Run: npm run scan
4. Copy folder to jd-partners-website/games/my-kid-3d-gallery/
5. git push → live at jdpartners.co/games/my-kid-3d-gallery/
```

`npm run scan` scans both artwork folders, compares against existing `manifest.json`, appends new entries with blank metadata fields, and leaves existing entries untouched.

## 3D Scene

### Camera & Character
- Third-person camera follows character from behind and slightly above
- Camera offset: ~3 units behind, ~2 units above character
- Click and drag rotates the camera around the character (orbit-style, no pointer lock needed)
- Character faces the direction of movement
- Walk animation plays when WASD/arrow keys are held; idle animation when still

### Character Model
- Source: Free low-poly humanoid from Mixamo (e.g. "Ybot" or similar)
- Format: `.glb` with embedded walk and idle animations
- Loaded via `THREE.GLTFLoader`
- Collision: character cannot walk through walls (simple AABB collision)

### Room Geometry
- Each room: box geometry — 4 walls + floor + ceiling, MeshStandardMaterial
- Doorways: opening cut into far wall, door frame trim
- Artwork frames: PlaneGeometry with golden frame border, image as texture
- All geometry generated procedurally in `rooms.js` from manifest data

## Artwork Interaction

### Proximity Detection
- Every frame, `artwork.js` checks the distance from the character to each artwork frame
- Threshold: **2.0 Three.js units** (approximately arm's length in scene scale)
- When character enters threshold:
  1. Nearest artwork is selected
  2. UI fades to black overlay
  3. Artwork image scales to fill the screen (centred, letterboxed)
  4. Character and gallery hidden
  5. Optional metadata shown below artwork (title, date, age) if fields are non-empty

### Exiting Fullscreen
- Press **ESC** to return immediately
- Walk backwards (character moves away) → auto-exit when distance exceeds threshold + 0.5 unit buffer (hysteresis to avoid flickering)

## Controls

### Desktop
| Input | Action |
|---|---|
| W / ↑ | Walk forward |
| S / ↓ | Walk backward |
| A / ← | Strafe left |
| D / → | Strafe right |
| Mouse drag | Rotate camera |
| ESC | Exit fullscreen artwork view |

### Mobile
- Virtual joystick (bottom-left): movement
- Touch drag anywhere else: rotate camera
- Implemented via `mobile.js` — detects touch device on load, renders joystick overlay

## Landing Page (`index.html`)

- Full-screen hero with gallery name and a brief description
- "Enter Gallery" button → navigates to `gallery.html`
- Simple, dark aesthetic matching the gallery interior
- No login, no authentication — fully public

## Out of Scope (v1)

- Multi-user / commercial version (separate project if pursued)
- Password protection or private access
- Backend / server-side file upload
- Comments or visitor interaction
- VR headset support
- Artwork ordering / drag-to-rearrange UI
